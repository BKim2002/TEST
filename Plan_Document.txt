1. GOALS

1.1  Practice Git version control
	- Understand the purpose of staging, committing, branching, and merging.

1.2  Practice GitHub workflow
	- Get comfortable creating, cloning, and pushing to a remote repository on GitHub.

1.3  Prepare for collaborative work
	- Understand pull requests and branch-based collaboration.

2. NEEDS

2.1  Software & Accounts
	- GitHub account
	- Git Desktop
	- IDE

2.2  Maintenance
	- Commit regularly with clear, descriptive messages.
	- Pull before starting new work to stay in sync.
	- Periodically delete merged branches to keep the repo tidy.

3. STEPS

	1. Repository Creation
		a. Log in to GitHub and click "New repository."
		b. Name the repo and add a short description (e.g., "Practice repo for learning Git").
		c. Check "Add a README file."
		d. Click "Create repository."
	2. Cloning
		a. Copy the repo URL from GitHub.
		b. Navigate into the repo folder
		c. Open README.md, add a sentence, and save.
		d. Stage the change.
		e. Commit with a message
		f. Push to GitHub.
	3. Branching & Merging
		a. Create and switch to a new branch.
		b. Add a new file with some content.
		c. Stage and commit.
		d. Switch back to main.
		e. Merge the branch.
		f. Push the updated main branch
		g. Delete the merged branch.
